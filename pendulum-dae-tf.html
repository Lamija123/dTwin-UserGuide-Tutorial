1.5. Pendulum Model Described by Differential-Algebraic Equations and Transfer Functions

Task Overview

This task involves modeling the motion of a mathematical pendulum using both system transfer functions and a set of differential-algebraic equations (DAE). The simulation includes changes in the position and velocity of a body attached to an inextensible string of constant length, with the option to introduce motion resistance.

Task objective:

TThe objective is to analyze the dynamic behavior of the pendulum by tracking changes in its position, velocity, and tension force over time.

Program Analysis:

Definition of the model under consideration:

Model [type=DAE domain=real method=Trapezoidal name="PendulumModelUsingDAE"]:

type = "DAE" – the system utilizes differential-algebraic equations (DAE).

name = "PendulumModelUsingDAE" – the model’s designated name.

domain = "real" – specifies that all variables and parameters belong to the real domain.

method = "Trapezoidal" – specifies the numerical integration method, in this case the trapezoidal method.

Block Vars:

Vars [out=true]: x=0.6 // Variable name and initial value (optional) v_x=0 // Variable name and initial value (optional) y=sqrt(l^2-x^2) // "0.998749218" Variable name and initial value (optional) v_y=0 // Variable name and initial value (optional) F=m*g*y/l // "9.6" Variable name and initial value (optional)

Block Params

Params: // default integration step l=1 // Parameter name and its value m=1 // Parameter name and its value g=9.81 // Parameter name and its value k=0 // koeficijent otpora zraka v [out=true]

The functions of the Vars and Params blocks were explained in example 1.

Block TFs:

TFs: x/v_x=1/s // First Transfer Function (speed along x-axis) y/v_y=1/s // Second Transfer Function (speed along y-axis)

This block is used to describe the system dynamics in the form of transfer functions within the Laplace domain, specifically using the operator s.

Block ODEs:

ODEs: v_x'=-F*x/(m*l)-k*v_x/m // Third state variable v_y'=g-F*y/(m*l)-k*v_y/m // 4th state variable

Block NLEs:

NLEs: x^2+y^2=l^2 // First nonlinear equation of the model

The functions of the NLEs and ODEs blocks were explained in examples 2 and examples 3, respectively. Here, differential and algebraic nonlinear equations describing the analyzed system are entered.

Block PostProc:

PostProc: v= v_x^2 + v_y^2

This block enables post-processing calculations of quantities, in this case the system’s velocity.

The keyword end signifies the termination of the model definition

end // end of Model

The previous and current models primarily differ in the numerical integration methods used and the way the system’s motion is modeled. The trapezoidal approach offers higher accuracy and stabi- lity, whereas the Runge-Kutta method enables faster, though somewhat less stable, simulations. The choice of method depends on the required precision and complexity of the system being modeled.
