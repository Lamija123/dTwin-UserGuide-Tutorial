Description of platform content

This section of the document provides a detailed overview of the fundamental commands within the dTwin platform and their usage. Additionally, it presents a set of functions available on the platform for solving mathematical problems across various domains.

Setup commands

Figure 1.2 displays the options available upon clicking the first icon (three parallel lines) located in the upper left corner of the interface. This action provides access to the core functionalities within the application.

The first option, Application, allows users to open the Settings window or exit the program via the Quit command.

Figure 1.2: View options in dTwin

By selecting the Settings command, a window will open, as shown in Figure 1.3, allowing the user to change the application language. The available language options are presented in a drop-down menu beneath the currently selected language. Additionally, there is an option that enables the display of icon names in the main window of the dTwin platform. This feature can be disabled by clicking the blue checkbox in the settings window.

Figure 1.3: Command Settings

Selecting another language and clicking on the Ok option requires restarting the application, picture 1.4. After that, it is possible to use dTwin in the desired language.

Figure 1.4: Change the language within the application

Modeling commands

The primary purpose of the dTwin platform is solving mathematical problems, specifically through system modeling. Therefore, one of the key commands in this context is the Model command.

Figure 1.5: Command Model

This command, illustrated in Figure 1.5, contains the following subcommands:

  • New Model – Creates a new blank model for further work. This command can be quickly activated using the shortcut Ctrl+N.
  • New document from template – Allows the creation of a new model using predefined templates. A variety of suggestions are offered to facilitate the modeling of typical systems.
  • Examples – Enables the user to browse and open existing model examples, which can be useful for learning and reference purposes.
  • Open model – Opens an existing model from disk. It can be accessed using the Ctrl+O shortcut.
  • Save - Saves the current model along with any changes made. This command is available only if a document is open and can be executed via Ctrl+S.
  • Save As – Allows the current model to be saved under a new name or in a different location, thereby preserving the original document.
  • Solve – This command is available only when a model or task is open. It initiates the solving process and provides both numerical and graphical (plot) outputs of the results.

An important part of the command Model is the Examples section, figure 1.6, which can serve as a foundation for learning and gaining a clearer understanding of the dTwin platform. These examples will be discussed in greater detail in the Tutorial chapter.

Figure 1.6: Examples within dTwin

Plot commands

Detailed usage of the plot and its features is demonstrated in the example located under Tutorial – General – Pendulum Model Described by Ordinary Differential Equations.

The Plot menu, figure 1.7, within the dTwin environment allows users to manage the display of simulation results in the form of graphs. These commands become active only when a model is opened and solved, i.e. when data is available for graphical representation (plotting). The menu includes the following options:

  • Full Screen – Enables full-screen display of the graph to facilitate easier viewing and analysis of results.
  • Show/Hide grid – Allows the user to toggle horizontal and vertical grid lines, which assist in reading values from the graph.
  • Show/Hide legend – Enables or disables the display of the legend, which identifies the quantities represented in the graph.
  • Save figure – Allows the current graph view to be saved as an image, which is useful for documentation and reporting purposes.
  • Go to the original figure position/zoom – Restores the graph to its initial view and zoom level, which is helpful if the graph has been moved or zoomed during analysis.
  • Adjust to figure settings – Enables detailed customization of the graph’s appearance, including color schemes, line styles, axis labels, and other graphical settings.

Main window commands

The main window of the dTwin application contains essential commands for working with models and tasks (as previously discussed). These commands (Figure 1.16) enable quick management of documents, model solving, and access to basic settings. The available commands include:

  • Settings – Opens the menu for configuring the application’s basic settings. Here, users can change the interface language and toggle the display of icon names in the main window.
  • New – Initiates the creation of a new blank model.
  • Template – Allows the user to open predefined modeling templates, which facilitate the creation of commonly used model structures.
  • Open – Used to open an existing model or task from the computer.
  • Save – Saves the currently opened model or task. This command is active only when a model or task is open.
  • Plot – Displays the results of a solved model in graphical form. This option becomes available only after a model has been opened and solved, and when data for graphical display is available.
  • Solve – Initiates the simulation of the active model. This option is available only when a model is open and allows for system behavior analysis as well as the generation of results and plots.
