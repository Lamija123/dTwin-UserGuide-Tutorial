Description of platform content

This section provides a demonstration of Unicode support in the dTwin platform, using Greek letters in model definitions as a valid part of variable and parameter names.

Unicode Example Model

This model example is accessible in dTwin via the path: Main Menu – Model – Examples – Unicode.

Model Description

The model is defined as a nonlinear model (type = NL) operating in the real domain, and is titled "Greek letters". Although the modeling language is English, both variable and parameter names are written using Greek letters, inserted using syntax e.g. \alpha renders as α. This illustrates the platform’s support for Unicode symbols in names, allowing more intuitive and professional mathematical notation.

Model Components

Vars Block:

Vars [out = true]: α // angle α [deg] β // angle β [deg]

Variables α and β represent angles in degrees. Their names use Unicode Greek letters, showing the system's ability to process non-ASCII characters.

Params Block:

Params: θ = 70 γ = 100

Parameters θ and γ are also named using Greek letters. Their values are used within the model's equations.

NLEs Block (Nonlinear Equations):

NLEs: 2*α + 2*β = θ 4*α + 2*β = γ

These equations define linear relationships between the variables and parameters. They demonstrate mathematical operations using Greek identifiers within the model structure.

PostProc Block:

PostProc:

No post-processing operations are defined for this model.

end // end of Model

Conclusion

This example demonstrates dTwin’s ability to support Unicode characters in model definitions. Using Greek letters such as α, β, θ, and γ as variable and parameter names improves the readability and clarity of mathematical models, especially in engineering and scientific applications. This Unicode feature allows users to adopt familiar symbolic notation and helps make models more intuitive and easier to interpret.
