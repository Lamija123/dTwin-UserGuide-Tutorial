Allowed Attributes and Their Types

Enumerations

Enumerations are fixed sets of named values used to define behavior or structure:

  • domain: real, cmplx, bool, int
  • type: NL, L, DAE, WLS, DC, AC, OPF
  • method: NR, GN, RK4, FD, TRAP, INV
  • signal: input, output, internal, none
  • backColor: auto, white, black, etc.

Real Values (float)

Floating-point attributes used for precision control, time steps, visual ranges, and mathematical operations:

  • eps: solution precision (e.g., 1e-6)
  • dT: time step used in dynamic simulations
  • min, max, width, step: define value ranges, graphical widths, or step sizes
  • alpha, beta, lambda, dev, mean: common constants used in equations or statistical measures
  • minX, maxX, minY, maxY: axis limits for plots

Integer Values (int)

Whole number attributes often used in iterations or logic control:

  • copyPars: Used in logic (e.g. copy parameters from submodels)
  • maxIter: Max allowed number of iterations in WLS, NR etc.

Boolean Values (bool)

Logical switches that enable or disable certain features:

  • out, conj, enabled, reInit, alwaysOn, initMain, legend, integer

String Values

Used for labeling and naming UI elements:

  • name: Name of model, variable, plot, etc.
  • xLabel, yLabel: Axis labels for plots

The image below shows how xLabel and yLabel are used in a linePlot element to define axis titles in the dTwin plotting environment:

Colors

Define plot colors for different UI themes:

  • colorL: Color in light mode
  • colorD: Color in dark mode

Line Patterns

Styling for line types in graphs:

  • pattern: solid, dash, dot, dashdot

All attributes must be used in accordance with their proper data types and within valid contexts.
