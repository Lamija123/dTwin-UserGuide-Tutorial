4.3. Dorf Example E.760 using TF model (hard disk)

The task that will be the subject of the analysis below is presented as example E.760 on page 481 in the textbook Modern Control Systems, by Richard C. Dorf and Robert H. Bishop.

This example considers the application of a PID controller to achieve a desired dynamic system response, utilizing velocity feedback to enhance control performance.

Task Overview

The assignment pertains to modeling a disk head positioning control system using a PD controller. The system represents a classic example of a closed-loop control system with negative feedback, aiming for precise control of the head's position based on a reference signal.

Task objective:

The objective of this assignment is to analyze the behavior of the control system in the presence of a reference signal and disturbances, and to determine how adjusting the parameters of the PD controller affects: the system's response speed, stability, precision in tracking the reference signal, and the ability to suppress the influence of disturbances. This type of model and analysis is essential in the design of precise control systems within mechatronics and automation.

Program Analysis:

Definition of the model under consideration:

Model [type=DAE domain=real method=RK4 name="Dorf Example 7.60 page 481"]:

type = ”DAE” – the system utilizes differential-algebraic equations.

name = ”Dorf Example 7.60 page 481” – model's name

domain = ”real” – defines that all variables and parameters are in the real domain.

method = ”RK4” – the fourth-order Runge-Kutta method is employed for solving.

Block Vars:

Vars: err [out=true] yc ym ul y [out=true]

Block Params

Params: u=1 [out=true] Kp=500 Kd=191.3 Td=0 // disturbance

The functions of the Vars and Params blocks were explained in example 1.

Block TFs:

TFs: yc/err=Kp+Kd*s // PD controller ym/yc=5000/(s+1000) // Motor coil y/ul=1/(s^2+20*s) // Load

The function of the TFs block is explained in examples 5.

Block NLEs:

NLEs: err=u-y // negative feedback ul=ym-Td // Input to load (output from motor minus disturbance)

The function of the NLEs block is explained in examples 2.

The keyword end signifies the termination of the model definition

end // end of Model
