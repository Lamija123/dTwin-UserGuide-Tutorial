3.1. RLC circut with Step Generator

Task Overview

This task involves modeling and analysis of a series RLC circuit excited by a step voltage function, utilizing an approach based on ordinary differential equations (ODEs). The model applies funda- mental physical laws governing electrical components — resistor (R), inductor (L), and capacitor (C) — to describe system behavior through key variables: input voltage (u_g), capacitor voltage (u_c), inductor current (i_L), and voltage drop across the inductor (u_L).

Task objective:

Model a series RLC system using a set of differential equations that describe the dynamic behavior of the components over time. The goal is to monitor the transient response of the circuit by analyzing the changes in current and voltage due to step excitation. Visualization of key variables (u_c, i_L, u_L) over time is essential for understanding the dynamics of the RLC circuit. Numerical results should be compared with theoretical expectations of the behavior of the RLC circuit under step excitation.

Program Analysis:

Definition of the model under consideration:

Model [type=ODE domain=real name="Serial RLC With Step Gen"]:

type = ”ODE” – the model solves ordinary differential equations.

name = ”Serial RLC With Step Gen” – model name

domain = ”real” – defines that all variables and parameters are in the real domain.

Block Vars:

Vars [out=true]: u_c=0 i_L=0

Block Params

Params: t=0 R=1 L=0.01 C=1e-5 u_g=10 [out=true] u_L=u_g [out=true]

The functions of the Vars and Params blocks were explained in example 1.

Block ODEs:

ODEs: i_L' = (u_g-u_c-R*i_L)/L u_c' = i_L/C

The Block a ODEs function is explained in examples 3. Here, differential and algebraic nonlinear equations describing the analyzed system are entered.

Block PostProc:

PostProc: u_L = u_g-u_c-R*i_L

After the main variables are simulated, the voltage across the inductor (L) is calculated, enabling additional visualization and verification of the inductor's state.

The keyword end signifies the termination of the model definition

end // end of Model
