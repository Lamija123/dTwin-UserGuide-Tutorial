Allowed functionalities in dTwin

The dTwin software enables the modeling and analysis of various mathematical and engineering systems. In order for users to fully utilize the capabilities of this tool, it is essential to understand the set of supported functionalities, functions, groups, and attributes that can be used in model descriptions.

The following section outlines all currently supported features, organized by category — mathematical functions, modeling groups, and attributes that define model parameters.

Allowed mathematical functions

Mathematical functions form the foundation for describing model behavior within dTwin. These functions can be used within expressions and equations that constitute the model. The functions are categorized into two groups: functions for operations with real numbers and functions for operations with complex numbers.

Real functions

The following functions are permitted in models using real numbers:

  • abs, acos, asin, atg, cos, exp, sqrt, ln, log, sin, tg
  • sqr, atg2, min, max, smin, smax, avg, var, dev, diff, sign
  • sinh, cosh, tgh, asnh, acsh, atgh
  • int, lim, rnd, disn, disc

Complex functions

The following functions are supported exclusively in models that involve complex numbers:

  • conj, cabs, rtop, ptor, real, imag

Allowed groups for digital model

The structure of a model in dTwin is organized into groups, each serving a specific role in the construction and execution of the model. Every group contains blocks that define variables, parameters, differential equations, and other model components.

The following groups are currently supported within the digital modeling framework:

  • Model, SubModel, Vars, Params, Distribs, Stats, ReInit, PreProc
  • NLEs, WLSEs, ECs, ODEs, PDEs, TFs
  • IterPostP, BasePostP, PostProc, Limits, Repeats, Plots

Allowed attributes and their types

Within each group and expression, various attributes may be used. Attributes represent parameters that further define the model’s behavior, and each has a strictly defined data type it must conform to.

The allowed attributes are as follows:

  • Enumerations: domain, type, method, signal, backColor
  • Real values (float): eps, dT, w, k, p, n, alpha, beta, lambda, min, max, mean, dev, step, width, minX, maxX, minY, maxY
  • Integers (int): copyPars, maxIter
  • Boolean values (bool): out, conj, enabled, reInit, alwaysOn, initMain, legend, integer
  • String values: name, xLabel, yLabel
  • Colors: colorL, colorD
  • Line patterns: pattern

All attributes must be used in accordance with proper syntax and their corresponding data types.
