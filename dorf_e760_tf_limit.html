4.4. Dorf Example E.760 using TF model with limit

Task Overview

In this assignment, a control system for disk head positioning was modeled, utilizing a PD controller. The model represents an extended version of the system from Example 7.60 in Dorf's book, and it is presented in the form of a differential-algebraic system (DAE) solved by the sixth-order Runge-Kutta method (RK6).

Task objective:

The objective of this assignment is to analyze the response of the disk head positioning system to a change in the reference signal under conditions both without and with disturbances, utilizing a PD controller with output limiting.

Program Analysis:

Definition of the model under consideration:

Model [type=DAE domain=real method=RK6 name="Dorf Example 7.60 str 481"]:

type = ”DAE” – the system utilizes differential-algebraic equations.

name = ”Dorf Example 7.60 page 481” – model's name

domain = ”real” – defines that all variables and parameters are in the real domain.

method = ”RK6” – the sixth-order Runge-Kutta method is employed for its solution.

Block Vars:

Vars: err [out=true] yc yclim [out=true] ym ul y [out=true]

Block Params

Params: u=1 [out=true] Kp=91.3 Kd=91.3 Td=0 // disturbance

The functions of the Vars and Params blocks were explained in example 1.

Block TFs:

TFs: yc/err=Kp+Kd*s // PD controller ym/yclim=5000/(s+1000) // Motor coil y/ul=1/(s^2+20*s) // Load

The function of the TFs block is explained in examples 5.

Block NLEs:

NLEs: err=u-y // negative feedback yclim=lim(yc, -300000, 300000) // limiter ul=ym-Td // Input to load (output from motor minus disturbance)

The function of the NLEs block is explained in examples 2.

The keyword end signifies the termination of the model definition

end // end of Model

In comparing the previous and this model of the disk head positioning control system, key differences are observed related to the simulation method, parameter selection, and additional functionality.

Beyond the varying PD controller parameter values, a notable distinction lies in the numerical integration method. The first model employs the fourth-order Runge-Kutta method (RK4), which is simpler and faster to execute but less precise compared to more complex methods. Conversely, the second model utilizes the sixth-order method (RK6), offering greater accuracy in simulation, particularly for more complex or nonlinear systems.

Furthermore, the introduction of a controller output limiter in the second model is a crucial difference. This serves as protection against control signal saturation and reflects the physical limitations of real-world actuators.

The currently analyzed model represents an improved and more realistic version of the system. By employing a more precise integration method, more moderate controller values, and incorporating a control signal limiter, this model better emulates the behavior of a real physical system, enabling a safer and more stable simulation.
