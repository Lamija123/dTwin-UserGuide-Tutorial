Allowed groups for digital model - explanation and examples

The following groups are currently supported within the digital modeling framework:

  • Modeling Groups in dTwin

    • Model

      • Defines the top-level model structure, including metadata such as model name, domain, accuracy, and solver type.

    • SubModel

      • Allows embedding of smaller models within a larger system to support modular design.

    • Vars

      • Declares the state variables of the system (e.g., x, y, z). These are typically unknowns to be solved.

    • Params

      • Declares fixed parameters that influence the model but are not solved for (e.g., constants).

    • Distribs

      • Defines probability distributions used in stochastic models.

    • Stats

      • Calculates statistical metrics such as mean, variance, standard deviation, etc.

    • ReInit

      • Allows resetting or reinitialization of variables during simulation runs.

    • PreProc

      • Contains preprocessing logic before the main simulation starts (e.g., data scaling, input formatting).

  • Equation-Based Groups

    • NLEs

      • Nonlinear equations group – used for systems of nonlinear algebraic equations.

    • WLSEs

      • Weighted Least Squares Estimators – for state estimation problems using measurement data.

    • ECs

      • Equality constraints that enforce algebraic conditions between variables or parameters.

    • ODEs

      • Ordinary Differential Equations – for modeling systems that evolve over time.

    • PDEs

      • Partial Differential Equations – for spatially distributed systems.

    • TFs

      • Transfer Functions – represent dynamic systems using Laplace-domain transfer functions.

  • Post-Processing and Visualization

    • IterPostP

      • Post-processing after each iteration (e.g., error computation, custom logging).

    • BasePostP

      • Base-level post-processing shared across all models or iterations.

    • PostProc

      • General post-processing logic after simulation ends.

    • Limits

      • Used to define upper and lower bounds on variables or parameters.

    • Repeats

      • Specifies how many times a simulation or part of it should be repeated (e.g., Monte Carlo runs).

    • Plots

      • Instructions for generating plots and visual representations of results.

Examples of how the aforementioned groups are used within a digital model can be found in the solved examples provided in the tutorial.
